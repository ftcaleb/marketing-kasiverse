## üîç Deep Dives

### Understanding the API Layer

**File**: `src/lib/api.js`

**Functions**:
1. `getNotes()` - Fetch all user notes
2. `createNote(data)` - Create new note
3. `updateNote(id, data)` - Update note
4. `deleteNote(id)` - Delete note
5. `getNoteById(id)` - Get single note
6. `isAuthenticated()` - Check auth status

**Features**:
- Auto-attach `Authorization: Bearer <token>` header
- Token from localStorage
- Error handling with redirects
- Network error detection
- 401 handling with redirect to /login

**See**: [QUICK_REFERENCE.md](QUICK_REFERENCE.md#api-functions)

### Understanding Component Patterns

**Patterns Covered**:
1. Create with API
2. Edit with inline form
3. Delete with confirmation
4. Error handling
5. Loading states
6. State updates

**See**: [QUICK_REFERENCE.md](QUICK_REFERENCE.md#component-patterns)

### Understanding State Management

**Before**: Redux store ‚Üí localStorage  
**After**: React state ‚Üê Backend API

**Benefits**:
- Single source of truth (backend)
- No Redux complexity
- Simpler code
- Easier testing

**See**: [IMPLEMENTATION_SUMMARY.md](IMPLEMENTATION_SUMMARY.md#state-management-flow)

### Understanding Data Flow

**Four Main Flows**:
1. Create - Form ‚Üí API ‚Üí Backend ‚Üí DB ‚Üí Frontend
2. Read - Page load ‚Üí API ‚Üí Backend ‚Üí DB ‚Üí Frontend
3. Update - Edit form ‚Üí API ‚Üí Backend ‚Üí DB ‚Üí Frontend
4. Delete - Confirmation ‚Üí API ‚Üí Backend ‚Üí DB ‚Üí Frontend

**See**: [DATA_FLOW_DIAGRAMS.md](DATA_FLOW_DIAGRAMS.md) for visual diagrams
